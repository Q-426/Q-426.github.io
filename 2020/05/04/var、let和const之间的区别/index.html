var、let和const之间的区别

1. var let const 声明时：

1. var的声明存在变量提升

2. let的声明不存在变量提升

3. const的声明不存在变量提升

2. var声明的变量会挂载在window上，而let和const声明的变量不会挂载在window上。

3. let和const声明会形成块作用域，var则不会;

典型例题:

为何同样的声明，为什么var和let的结果不同呢？
因为let相对于var会形成块级作用域；
let出现在类似for循环时，会有特殊的表现；js引擎会为for后面跟的大括号每次都创建一个词法环境，将let声明的变量每次都拷贝一份都这个词法环境中，也就是说每次执行的for的函数体里的i，都是一个同名的副本，所以每次setTimeout中绑定的号也就是那个同名副本

4. 同一作用域下let和const不能声明同名变量；var则可以；

5. js在扫描发现变量声明时，要么将他们提升到作用域顶部（遇到var声明）;要么将声明放在临时性死区（TDZ）中（遇到let和const 声明）;

当前块级作用域中存在a使用let/const声明的情况下，给a赋值10时，只会在当前作用域中找变量a,而此时，还未到let声明时，所以会
Error:a is not defined;
此时a存在于TDZ中;
